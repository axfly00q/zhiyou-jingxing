# 部署与使用手册

## 一、硬件 / 软件要求

| 项 | 推荐 | 最低 |
|---|---|---|
| 系统 | Ubuntu 22.04 / Windows 11 + WSL2 | 同 |
| 内存 | 16 GB | 8 GB |
| 显存 | NVIDIA 24 GB（4090，跑 LiveTalking + CosyVoice2） | 8 GB（仅文本对话，无数字人） |
| Docker | 24+ + docker compose v2 | — |
| Node.js | 20+（仅本地开发） | — |
| Python | 3.11（仅本地开发） | — |

## 二、目录结构
```
zhiyou-jingxing/
├── orchestrator/        # 自研 FastAPI 编排层
├── frontend-tourist/    # 游客端
├── frontend-admin/      # 运营后台
├── frontend-dashboard/  # 数据大屏
├── deploy/              # docker-compose、.env
└── docs/                # 文档
```

## 三、外部依赖准备

下列重型组件**不**在本仓库 docker-compose 内启动，请按各自官方文档单独部署，
然后把 URL 写入 `deploy/.env`：

| 组件 | 文档 | 默认端口 |
|---|---|---|
| Dify（LLM/RAG 中台） | https://github.com/langgenius/dify | 5001 |
| LiveTalking（数字人） | https://github.com/lipku/LiveTalking | 8010 |
| FunASR runtime | https://github.com/modelscope/FunASR | 10095 |
| CosyVoice2 | https://github.com/FunAudioLLM/CosyVoice | 8001 |

> 评审/演示如缺其中某项，对应能力会自动降级（例如未配置 LiveTalking 时
> 游客端会显示数字人占位图，仍可走文本/语音问答）。

## 四、首次启动

```bash
git clone <repo> zhiyou-jingxing
cd zhiyou-jingxing
cp deploy/.env.example deploy/.env
# 编辑 deploy/.env：填 DEEPSEEK_API_KEY、DIFY_API_KEY 等
cd deploy
docker compose up -d --build
```

启动后：
- 游客端  http://localhost:5173
- 后台    http://localhost:5174  （admin / admin123）
- 大屏    http://localhost:5175
- API     http://localhost:8000/docs

## 五、知识图谱初始化

```bash
docker compose exec orchestrator python -m scripts.build_kg
```
脚本读取 `orchestrator/data/kg/*.json` 写入 Neo4j；
无 Neo4j 时，`kg_planner` 会**自动**回退到 JSON 直接读取，无需此步。

## 六、知识库导入到 Dify
1. 浏览器访问 Dify 控制台（默认 http://localhost）。
2. 创建一个"聊天助手"应用 → 关联一个新知识库。
3. 把 `orchestrator/data/kb/*.md` 里的内容上传到该知识库。
4. 调整分块（建议 400 字 + 50 字重叠）+ 开启 Rerank。
5. 在应用 Prompt 模板中加入：
   > "你是苏州园林的资深导游数字人。请基于知识库回答游客问题，引用出处。"
6. 复制应用的 **API Key**，填入 `deploy/.env` 的 `DIFY_API_KEY`，重启 orchestrator。

## 七、数字人形象与音色配置

1. 部署 LiveTalking 后，将自带形象编号（如 `guzhuang_female_01`）填入：
   - `deploy/.env`  →  `LIVETALKING_DEFAULT_AVATAR`
2. 在 CosyVoice2 中预录 1-2 段景区导游词样音 → 生成 `voice_id`，填入：
   - `deploy/.env`  →  `TTS_DEFAULT_VOICE`
3. 在运营后台 `数字人形象` 页新增形象记录，将 `code` 与 `voice_id` 关联。

## 八、本地开发模式

```bash
# 后端
cd orchestrator
python -m venv .venv && .venv\Scripts\activate
pip install -e .[dev]
uvicorn app.main:app --reload --port 8000

# 前端（任一）
cd ../frontend-tourist && npm install && npm run dev
```

## 九、运行测试

```bash
cd orchestrator
pytest
```
当前覆盖核心路线规划与情感分析规则兜底逻辑；
端到端测试需先启动 Dify/LiveTalking/FunASR/CosyVoice2。

## 十、常见问题

- **数字人画面不显示**：`LIVETALKING_BASE_URL` 未配置或浏览器 WebRTC 被拦截。
- **LLM 不出声**：检查 Dify API Key 与应用是否正确绑定知识库。
- **TTS 沉默**：检查 CosyVoice2 服务可用性（`curl ${TTS_BASE_URL}/api/v1/tts ...`）。
- **建议引擎不出建议**：需要 ≥ 5 条情感为 neg 的消息才会触发；可手工触发 `python -m app.services.scheduler`。
