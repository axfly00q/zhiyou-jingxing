# 总体设计文档

## 1. 设计目标
利用 AI 数字人技术，构建一套面向苏州园林（拙政园 / 留园）的智能、互动、个性化导览
服务系统，覆盖游客交互端、管理后台、数据大屏三端，对应大赛"功能完整度 / 技术与
创新性 / 行业实用性 / 文档质量"四个评分维度。

## 2. 核心策略：复用 + 自研

| 模块 | 选型 | 性质 |
|---|---|---|
| 数字人渲染 | three-vrm + VRM 1.0（浏览器本地） | 复用库 + 自研集成 |
| 嘴型同步 | Web Audio API 频谱分析 → 5 元音权重 | **自研** |
| 表情/动作标签 | 后端关键词规则→VRM ExpressionManager | **自研** |
| LLM / RAG 知识库 | Dify | 复用 |
| ASR | FunASR | 复用 |
| TTS | CosyVoice2 | 复用 |
| 数字人形象素材 | VRoid Studio / VRoid Hub 开放模型 | 复用 |
| **对话编排中间层** | FastAPI（自研） | **自研** |
| **知识图谱 + 个性化路线规划** | Neo4j + 自研贪心算法 | **自研（创新点 1）** |
| **情感分析 + 服务建议引擎** | LLM + APScheduler | **自研（创新点 2）** |
| 三个前端 | Vue3 + Vite | 自研 |

理由：**底层引擎复用** 保证演示效果稳定（口型、检索、TTS 都是研究级难题），
**业务编排与亮点模块自研** 拉创新分与差异化体验。

```
                  HTTP │   WS    │
                       ▼         ▼
              ┌────────────────────────────┐
              │  Orchestrator (FastAPI)     │
              │  ┌─────────────────────┐   │

              │  │ /route (路线规划)    │   │
              │  │ /analytics (大屏)   │   │
              │  │ /admin (后台)       │   │
              │  └─────────────────────┘   │
              └─┬─────┬─────┬─────┬─────┘
                │     │     │     │
        Dify  ──┘     │     │     │
        FunASR ───────┘     │     │

```

> 数字人画面不再依赖服务端 WebRTC 推流：浏览器加载后台上传的 `.vrm` 模型，
> 用 [VrmAvatar.vue](../frontend-tourist/src/components/VrmAvatar.vue) +
> [lipSync.js](../frontend-tourist/src/lib/lipSync.js) 在本地渲染，零 GPU 服务依赖。

## 4. 关键数据流

### 4.1 一次完整对话
1. 游客在 [Chat.vue](../frontend-tourist/src/views/Chat.vue) 输入文字 / 录音；
2. 录音 → `POST /api/chat/voice` → FunASR 转写 → 文本；
3. 文本入库（[Conversation/Message](../orchestrator/app/models/__init__.py)），同时异步派发 `analyze()` 做情感/意图打标；
4. `Dify.chat()` 执行 RAG 知识问答，返回 `answer + citations`；
5. `CosyVoice2.synthesize()` 合成 MP3，base64 内联返回前端播放；
6. 同时后端 [`dialogue_tagger.tag()`](../orchestrator/app/services/dialogue_tagger.py) 从回复文本
   推出 `(emotion, motion)` 随响应返回；
7. 前端 [VrmAvatar.vue](../frontend-tourist/src/components/VrmAvatar.vue) 收到 `audio_url`
   后本地播放并用 Web Audio FFT 驱动嘴型；emotion/motion 则切换 BlendShape
   与 `.vrma` 动作 clip；
8. 游客点"打断"按钮 → 前端 `audioEl.pause()`，嘴型/表情自动复位。

首字延迟目标 ≤ 3s（Dify blocking 模式 ~1.5s + TTS ~0.8s + 前端 ~0.3s）。

### 4.2 个性化路线规划（创新点 1）
1. 游客在 [Preference.vue](../frontend-tourist/src/views/Preference.vue) 设置 5 维偏好滑块 + 时长；
2. `POST /api/route/{park}/plan` 触发 [`kg_planner.plan_route()`](../orchestrator/app/services/kg_planner.py)：
   - 从 Neo4j（或 JSON 回退）读 `ParkGraph`；
   - 计算每个景点的 `theme_score = Σ pref[k] × spot.themes[k]`；
   - 以最高分景点为起点贪心扩张：`next_score = theme_score - α × walk_minutes`；
   - 累计耗时（讲解 + 步行）逼近 `duration_min` 时停；
3. LLM 根据偏好与路线生成口语化开场白（80 字内）；
4. 前端展示路线卡片，"让数字人讲解"跳转到 [Chat.vue](../frontend-tourist/src/views/Chat.vue)，
   开场白被首条消息播报。

### 4.3 情感分析 + 服务建议（创新点 2）
1. 每条用户消息异步进入 [`sentiment.analyze()`](../orchestrator/app/services/sentiment.py)：
   - LLM JSON 模式输出 `{intent, sentiment, sentiment_score, keywords}`；
   - 失败兜底走关键词规则；
2. APScheduler 每 30 分钟运行 [`aggregate_suggestions_job`](../orchestrator/app/services/scheduler.py)：
   - 取近 24h `sentiment=neg` Top30 消息；
   - LLM 提炼"问题摘要 + 改进建议 + 优先级"，写入 [`Suggestion`](../orchestrator/app/models/__init__.py) 表；
3. 后台 [Suggestions.vue](../frontend-admin/src/views/Suggestions.vue) 与大屏均消费此表。

## 5. 数据模型（PostgreSQL）

| 表 | 关键字段 | 说明 |
|---|---|---|
| `conversation` | session_id, tourist_profile, avatar_id | 一次会话 |
| `message` | role, content, intent, sentiment, sentiment_score, keywords, citations, latency_ms | 每条消息 |
| `avatar` | code, name, voice_id, is_default | 形象与音色绑定 |
| `suggestion` | title, summary, priority, evidence, status | LLM 生成的建议 |

## 6. 知识图谱模型（Neo4j）

```
(Park {code, name})
  -[:HAS_SPOT]->
(Spot {code, name, themes JSON, highlight, suggested_minutes})
  -[:NEXT_TO {walk_minutes}]->
(Spot)
```

`themes` 5 维：history / nature / architecture / family / photo，每维 0~1。
当前预置：拙政园 11 个节点 / 留园 11 个节点（[orchestrator/data/kg](../orchestrator/data/kg/)）。

## 8. 安全
- JWT（HS256）保护后台写接口；游客对话与大屏接口公开只读。
- 所有第三方 API Key 仅存于 `.env`，已加入 `.gitignore`。
- LLM 输出未直接执行，仅作为文本/JSON 内容；JSON 模式 + Pydantic 严校验。
- CORS 白名单限定三个前端域名。

## 9. 演进路线
- 短期：补充更多园林（虎丘 / 寒山寺）、英文/苏州话方言。
- 中期：移动端小程序、AR 实景叠加讲解。
- 长期：跨景区知识图谱联邦、多游客协作游览模式。
